# Contributing to Solar System Sim 25

We welcome contributions to improve the Solar System Simulator! To ensure a smooth process, please follow these guidelines.

## How to Contribute

### Reporting Issues
- If you find a bug or have a feature request, please [open an issue](https://github.com/therealsahil19/solarsystemsim25/issues) on GitHub.
- Please check existing issues to avoid duplicates.
- Provide as much detail as possible, including steps to reproduce the bug or a clear description of the feature.

### Making Changes
1.  **Fork the repository** to your own GitHub account.
2.  **Create a new branch** for your changes. Please use a descriptive branch name.
    -   `feat/...` for new features (e.g., `feat/add-kuiper-belt`)
    -   `fix/...` for bug fixes (e.g., `fix/camera-zoom-issue`)
    -   `chore/...` for maintenance tasks (e.g., `chore/update-dependencies`)
3.  **Make your changes** in your branch.
4.  **Test your changes** locally to ensure they work as expected.
5.  **Open a Pull Request (PR)** to the `main` branch of this repository.
    -   Provide a clear title and description for your PR.
    -   Link to any relevant issues.

## Branching and PR Strategy

We follow a "one branch, one PR per logical task" model. This helps keep PRs small, focused, and easy to review.

- **Create one focused branch per logical task.** This keeps PRs small, reviewable, and easy to revert if something breaks.
- Make each branch target `main` (or the default branch).
- Your PR will be reviewed, and once approved, it will be merged.

Thank you for your contributions!
